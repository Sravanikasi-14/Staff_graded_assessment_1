Question 4 - File Access and I/O Investigation

Aim:
To understand file access and input/output operations in Linux.

Open Files:
The open files and file descriptors were checked using Linux commands.

File Descriptors:
The file descriptor information showed how Linux manages opened files for the current process.

Output Redirection:
The output of a command was redirected to output.txt using the '>' operator.
Another line was added using the '>>' operator.

Error Redirection:
The error generated by an invalid command was redirected to error.txt using '2>'.

Resource Limits:
The ulimit command displayed the system resource limits for the current user.

Conclusion:
This experiment helped in understanding file access, file descriptors, output redirection, error redirection, and resource limits in Linux.
